# AutoYoloTrainer 실행 가이드

## 1. 권장 환경
- Windows 10/11
- Python 3.10 이상
- NVIDIA GPU 사용 시 NVIDIA Driver 설치 필요

## 2. 의존성 설치
프로그램 폴더에서 아래 파일을 실행합니다.

```text
install_dependencies.bat
```

수동 설치가 필요하면 `Code` 폴더에서 실행합니다.

```powershell
py -3 -m pip install -r requirements.txt
```

## 3. 실행
빌드된 실행 파일:

```text
AutoYoloTrainer.exe
```

소스 실행:

```powershell
cd Code
python main.py
```

## 4. 기본 학습 순서
1. 프로그램 실행
2. 하드웨어 환경 확인
3. 학습 모델 `.pt` 선택 또는 다운로드
4. 학습 프로젝트명 입력 또는 기존 프로젝트 선택
5. 원본 데이터셋 폴더 불러오기
6. `Train / Val / Test` 비율 설정
7. `복사` 또는 `이동` 실행
8. 학습 옵션 입력
9. `학습 시작` 실행
10. 중앙 로그와 학습 상태 확인

## 5. 데이터셋 입력 조건
원본 데이터셋 폴더에는 다음 파일이 필요합니다.

```text
classes.txt
image001.jpg
image001.txt
image002.jpg
image002.txt
```

이미지와 같은 이름의 `.txt` 라벨이 없으면 제외됩니다.

## 6. 생성되는 프로젝트 구조
```text
Work/
  ProjectName/
    classes.txt
    data.yaml
    images/
      train/
      val/
      test/
    labels/
      train/
      val/
      test/
    result/
```

이미 내용이 있는 같은 이름의 프로젝트에는 새 데이터셋을 추가하지 않습니다.

## 7. 모델 변환
우측 `모델 변환` 영역에서 진행합니다.

1. `PT 선택`
2. 변환 형식 선택
3. `Export`

export 결과는 기본적으로 `Work` 아래에 저장됩니다. Ultralytics 형식별 정책에 따라 하위 폴더가 생성될 수 있습니다.

## 8. 이미지 디텍팅
우측 `이미지 디텍팅` 영역에서 진행합니다.

1. `.pt` 모델 선택
2. 이미지 선택
3. `디텍팅`
4. 결과 이미지와 검출 클래스 / 정확도 확인

이미지는 뷰어 크기에 맞춰 원본 비율을 유지하며 확대 또는 축소됩니다.

## 9. 주의사항
- `추가 학습`은 아직 실제 resume 기능이 구현되지 않았습니다.
- GPU가 표시되어도 PyTorch CUDA가 `False`이면 GPU 학습이 되지 않습니다.
- 학습 중 콘솔 창이 반복 표시되면 `Workers`가 0인지 확인합니다.
- `yolo11n.pt`가 학습 결과 폴더에 자동 생성되지 않도록 `amp=False`가 적용되어 있습니다.

## 10. EXE 빌드
`Code` 폴더에서 실행합니다.

```powershell
.\build_exe.ps1
```

기존 exe가 교체되지 않으면 실행 중인 프로그램을 종료한 뒤 다시 빌드합니다.
