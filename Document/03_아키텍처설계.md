# AutoYoloTrainer 아키텍처 설계

## 1. 구성
AutoYoloTrainer는 `PyQt6 GUI`, 서비스 계층, 외부 Python 러너로 구성됩니다.

```text
main.py
  autoyolotrainer/app.py
    ui/main_window.py
    ui/dialogs.py
    ui/widgets.py
    services/
      dataset_service.py
      hardware_service.py
      model_service.py
      software_service.py
      training_worker.py
  training_runner.py
  model_tool_runner.py
```

## 2. UI 계층
- `main_window.py`
  - 좌측 설정 패널
  - 가운데 로그 / 학습 정보 / 학습 상태
  - 우측 모델 변환 / 이미지 디텍팅 / 검출 결과
- `widgets.py`
  - 토글 버튼
  - 학습 진행 그래프 위젯
- `styles.py`
  - 다크 / 라이트 스타일시트

## 3. 서비스 계층
- `dataset_service.py`
  - 데이터셋 스캔
  - train / val / test 분할
  - `data.yaml` 생성
- `hardware_service.py`
  - CPU, GPU, RAM, 저장소 감지
  - NVML 기반 GPU 사용량 조회
- `model_service.py`
  - YOLO 모델명 파싱
  - 기본 모델 다운로드
- `training_worker.py`
  - UI 스레드와 학습 프로세스 분리
  - 학습 로그와 진행률 전달

## 4. 외부 러너
- `training_runner.py`
  - Ultralytics YOLO 학습 실행
  - `amp=False` 적용
  - `AUTO_YOLO_TRAINER_PROGRESS` 로그 출력
  - `result/yolo11n.pt` 자동 생성 시 제거
- `model_tool_runner.py`
  - 모델 export 실행
  - 이미지 detect 실행
  - 결과를 JSON으로 반환

## 5. 데이터 흐름
1. 사용자가 데이터셋 폴더 선택
2. `dataset_service.py`가 유효 이미지 / 라벨 쌍 스캔
3. `Work/<프로젝트명>` 아래 YOLO 구조 생성
4. 사용자가 모델과 학습 옵션 선택
5. `TrainingWorker`가 별도 Python 프로세스로 `training_runner.py` 실행
6. 러너 출력 로그를 UI가 수신
7. 학습 완료 후 결과는 `Work/<프로젝트명>/result`에 저장

## 6. 모델 변환 / 디텍팅 흐름
1. 우측 패널에서 `.pt` 선택
2. export 또는 detect 실행
3. `model_tool_runner.py`를 별도 Python 프로세스로 실행
4. export 결과 경로 또는 detect 결과 JSON 수신
5. UI에 결과 이미지와 검출 라벨 표시

## 7. 경로 정책
- 실행 루트: exe 위치
- 설정: `Data/settings.json`
- 다운로드 모델: `Data/models`
- Ultralytics 설정: `Data/ultralytics`
- 학습 프로젝트: `Work`
- export 파일 기본 출력: `Work`
- 이미지 선택 기본 위치: `Work`

## 8. 설계 원칙
- UI와 장시간 작업 분리
- 사용자 데이터와 설정 데이터 분리
- 자동 생성 파일은 예측 가능한 위치에만 생성
- 사용자가 직접 선택한 `.pt` 경로는 제한하지 않음
