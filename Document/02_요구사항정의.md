# AutoYoloTrainer 요구사항 정의

## 1. 프로그램 환경
- 프로그램명 표시
- 설정 창 제공
- 다크 / 라이트 테마 전환
- 설정값을 `Data/settings.json`에 UTF-8로 저장
- 프로젝트 루트 이동 후에도 실행 파일 기준 경로 사용

## 2. 하드웨어 표시
- CPU 이름 표시
- GPU 이름과 메모리 표시
- RAM 크기 표시
- 저장소 사용량 표시
- 학습 중 앱 / 학습 프로세스 CPU, 메모리 표시
- GPU 사용량은 `nvidia-smi` 호출 없이 NVML로 조회

## 3. 모델 관리
- 학습용 YOLO `.pt` 파일 직접 선택
- 번호 / 크기 드롭다운 기반 기본 모델 다운로드
- 다운로드 모델 저장 위치는 `Data/models`
- 사용자가 직접 선택하는 `.pt`는 Work/result 등 다른 위치도 허용
- 우측 패널에서 export / detect용 `.pt` 선택 가능

## 4. 데이터셋 처리
- 입력 데이터셋은 `classes.txt`와 이미지 / 라벨 쌍 기준
- 이미지와 같은 이름의 `.txt` 라벨이 있어야 유효 데이터로 판단
- 유효하지 않은 파일은 제외
- `Train / Val / Test` 비율 합계는 100이어야 함
- 셔플 여부 선택 가능
- 복사 또는 이동 방식 선택 가능
- 결과 구조는 YOLO 표준 구조 사용

## 5. 학습 프로젝트
- 프로젝트 폴더는 `Work/<프로젝트명>` 아래 생성
- 이미 내용이 있는 같은 이름의 프로젝트에는 새 데이터셋 분할을 추가하지 않음
- 기존 프로젝트는 버튼으로 선택 가능
- 학습 결과는 프로젝트 내부 `result` 폴더에 저장

## 6. 학습 실행
- 백그라운드 Python 프로세스로 YOLO 학습 실행
- UI 프리징 방지
- 진행률, 에포크, 자원 사용량 표시
- 학습 중 콘솔 창이 뜨지 않아야 함
- Windows에서는 `Workers`를 0으로 조정
- AMP 자동 검사로 `yolo11n.pt`가 생기지 않도록 `amp=False` 적용

## 7. 우측 패널
- `.pt` 파일 선택
- export 형식 선택
- ONNX, OpenVINO, TensorRT engine, TorchScript, TFLite 변환 버튼
- 이미지 파일 선택
- 선택 모델로 이미지 디텍팅
- 결과 이미지 표시
- 검출 클래스와 정확도 표시
- 이미지 뷰어는 영역 크기에 맞춰 원본 비율 유지 확대 / 축소

## 8. UI 요구사항
- 가운데 로그는 세로 스크롤만 허용
- 로그 아이템 내부 여백은 작게 유지
- 가운데 로그 / 학습 정보 영역은 세로 Splitter로 크기 조절 가능
- 우측 모델 변환 / 이미지 디텍팅 영역은 세로 Splitter로 크기 조절 가능

## 9. 배포 요구사항
- `AutoYoloTrainer.exe` 단일 실행 파일 제공
- 학습과 export는 시스템 Python 환경의 패키지를 사용
- 다른 PC 이동 시 `install_dependencies.bat`로 의존성 설치 가능
