# AutoYoloTrainer 프로젝트 개요

## 1. 목적
AutoYoloTrainer는 YOLO 학습 준비, 실행, 결과 확인, 모델 변환, 이미지 디텍팅을 한 화면에서 처리하기 위한 Windows용 GUI 도구입니다.

핵심 목적은 다음과 같습니다.
- 현재 PC의 CPU, GPU, RAM, 저장소 상태 확인
- YOLO `.pt` 모델 선택 또는 다운로드
- 원본 데이터셋을 YOLO 학습 구조로 자동 분할
- 백그라운드 학습 실행과 진행 상태 확인
- 학습된 `.pt` 모델을 ONNX 등으로 변환
- 선택 이미지에 대해 모델 디텍팅 결과 확인

## 2. 기본 폴더 구조
```text
AutoYoloTrainer/
  AutoYoloTrainer.exe
  install_dependencies.bat
  Code/
  Data/
    models/
    settings.json
    ultralytics/
  Document/
  Work/
```

## 3. 주요 기능
- 다크 / 라이트 테마
- 설정값 `Data/settings.json` 저장
- YOLO 모델 다운로드 경로 `Data/models`
- 학습 프로젝트 경로 `Work/<프로젝트명>`
- 데이터셋 복사 / 이동
- `images/train`, `images/val`, `images/test` 자동 구성
- `labels/train`, `labels/val`, `labels/test` 자동 구성
- `classes.txt`, `data.yaml`, `result` 자동 생성
- 학습 로그 세로 스크롤 표시
- 학습 진행률 및 에포크 표시
- 학습 중 CPU, 메모리, GPU 사용량 표시
- 우측 패널에서 `.pt` 선택, export, 이미지 디텍팅

## 4. 안정성 관련 반영 사항
- 학습 프로세스는 별도 Python 프로세스로 실행
- Windows GUI 환경에서 콘솔 창이 반복 표시되지 않도록 숨김 실행
- Windows 학습 안정성을 위해 DataLoader `Workers`는 0으로 조정
- GPU 사용량 조회는 `nvidia-smi` 대신 NVML 기반으로 처리
- Ultralytics AMP 검사로 `yolo11n.pt`가 생성되는 문제를 막기 위해 `amp=False` 적용
- 그래도 `result/yolo11n.pt`가 생기면 학습 종료 후 해당 파일만 제거

## 5. 현재 미구현
- 추가학습 실제 resume 로직
- 기존 프로젝트에 추가 데이터셋을 별도 등록해 함께 학습하는 기능
- 추가 데이터셋 목록 관리 UI
- 손실값, mAP 등 상세 학습 그래프

## 6. 주의
현재 `추가 학습` 버튼은 UI 상태 저장용이며 실제 추가학습 기능은 아직 구현되지 않았습니다.
